회사 통합 시스템 기획 초안

화면구성안 + 메뉴구조도 + 데이터 통합 방향 (MVP 기준)

1) 메인 진입 화면 (4개 사각 메뉴)

경영진
임원
본사
현장
  • 경영진/임원: 별도 인증(2차 인증 권장) 후 전용 메뉴로 이동
  • 본사: 부서별 자동화 시스템 허브로 이동
  • 현장: 입퇴거/민원/공지 등 현장 운영 메뉴로 이동

2) 권한별 주요 메뉴 구조

권한핵심 메뉴비고
경영진사업수지, 전사 KPI, 경영 리포트현재 개발된 사업수지 시스템 우선 연결
임원사업수지, 담당부문 실적, 승인/결재부문별 Drill-down
본사(기획관리부)입퇴거 관리, 입주민 입주 설문조사부서 자동화 1순위
현장입퇴거 현황, 설문, 민원/하자, 공지모바일 접근 고려

3) 데이터 통합/조인 방향

공통 키연결 테이블(예시)활용 지표
project_id / site_idbusiness_balance + move_in_out + surveys현장별 손익/입주율/만족도
department_idusers + permissions + workflows부서별 업무 처리량/리드타임
resident_idresidents + survey_answers + move_in_out입주여정 분석

4) MVP 단계 제안

  • 통합 포털(4메뉴) + 권한 로그인
  • 경영진/임원: 사업수지 연결
  • 본사 기획관리부: 입퇴거 + 입주 설문 연결
  • 공통 코드(현장/프로젝트/부서) 정리 후 DB 조인 확장
